第1章

Claude Code とは何か

これは何のツール？

Claude Codeは、Anthropic社が開発したAIアシスタントです。ChatGPTのような対話型AIですが、決定的に違う点が一つあります。あなたのパソコンの中のファイルを直接読み書きできるということです。

たとえば、ChatGPTでは「この文章を修正して」と頼んだら、修正後のテキストが画面に表示されるので、自分でコピーして貼り付ける必要がありました。Claude Codeでは、「この書類を修正して」と頼むだけで、AIが直接ファイルを開き、修正し、保存してくれます。

つまり、これまで「AIに聞く → 答えを見る → 自分で作業する」だった流れが、「AIに頼む → AIが作業する → 結果を確認する」に変わります。

なぜミラツクで使うのか

私たちの業務には、AIが得意とする作業がたくさんあります。

AI が得意な作業具体例
文書の整理・構造化リサーチ結果のまとめ、議事録の作成
ドラフト作成報告書、提案書、メールの下書き
データの集計・分析アンケート集計、傾向分析
翻訳・要約英語文献の翻訳、長文の要約
アイデア出し企画のブレスト、構成案の複数パターン作成

これらの作業をAIに任せることで、私たちは「考える仕事」「人と対話する仕事」——つまり、ミラツクの本質的な価値である対話と探究に、より多くの時間を使えるようになります。

最初の一歩

ステップ0: ツールを起動する

Claude Codeは Cursor というアプリの中で使います。Cursorは、ChatGPTのようなチャット画面でAIとやりとりできるアプリです。ターミナル（黒い画面）を使う必要はありません。

  1. Cursorアプリを開く
  2. フォルダを開く（自分の作業フォルダを選択）
  3. 画面右側のチャットパネルでAIに話しかける

ステップ1: 最初の成功体験を得る

まずは「これは便利だ」と実感することが大切です。以下から、自分の業務に近いものを一つ選んで試してください。

4ステップ思考プロトコル

AIに何かを頼むときは、この4ステップを意識してください。これを習慣にするだけで、結果の質が大きく変わります。

AIへの依頼の基本形

STEP 1
調査
まず現状を把握させる
「このフォルダの中身を確認して」
「この文書の構成を教えて」
STEP 2
計画
やることを整理させる
「どう進めるか計画を立てて」
「必要な手順をリストアップして」
STEP 3
実行
作業を依頼する
「では、その計画に沿って進めて」
「まず最初のステップだけやって」
STEP 4
確認
結果を検証させる
「内容を読み返して問題ないか確認して」
「抜け漏れがないかチェックして」
特に重要なのは「調査」と「確認」

いきなり「実行」から始めると、AIが状況を誤解したまま作業を進めることがあります。また、AIの出力は必ず自分の目で確認してください。AIは事実と異なることをもっともらしく書くことがあります（ハルシネーション）。

第4章

CLAUDE.md を設定する

CLAUDE.md とは

CLAUDE.mdは、AIが作業を始める前に必ず読む「業務マニュアル」のようなファイルです。ここに組織のルールや自分の役割を書いておくと、毎回説明し直す手間がなくなります。

たとえば、CLAUDE.mdに「報告書はですます調で書く」と書いておけば、毎回「ですます調で書いて」と指示する必要がありません。

設定方法

  1. 自分の作業フォルダの一番上の階層に CLAUDE.md というファイルを作る
  2. 以下のテンプレートをコピーして貼り付ける
  3. 【】の部分を自分の情報に書き換える
CLAUDE.md テンプレート（コピーして使ってください）
# 業務コンテキスト

## 自分について
- 名前: 【あなたの名前】
- 役割: 【例: リサーチアシスタント、事業推進担当】
- 主な業務: 【例: ワークショップの企画運営、報告書作成】

## 組織について
- 組織名: NPO法人ミラツク / 株式会社エッセンス
- 事業内容: 社会課題に取り組む多様なステークホルダーの
  対話と協働を促進
- 大切にしている価値: 対等な立場での探究、多様な知の統合

## 文章のルール
- 報告書: ですます調、地の文中心（箇条書きだけにしない）
- メール: 丁寧だが堅すぎないトーン
- 「参加者」より「ご参加くださる皆さま」を好む

## 注意事項
- 個人情報・機密情報は扱わない
- 事実と推測は明確に区別する
- 分からないことは「分かりません」と正直に答える
CLAUDE.md に書いてはいけないもの

パスワード、APIキー、クライアントの機密情報、個人の住所・電話番号、銀行口座・カード番号。CLAUDE.mdはテキストファイルなので、見られても問題ない情報だけを書いてください。

第5章

よくあるトラブルと対処法

Q1: AIの回答がだんだんおかしくなってきた

原因: AIには「一度に覚えていられる量」に限界があります。

ノートをイメージしてください。AIとの会話は、ノートにどんどん書き足していくようなものです。ノートのページがいっぱいになると、最初の方に書いたことが読めなくなります。これと同じことがAIにも起こります。

対処法

長い会話が続いたら、新しいセッション（会話）を始める。一つの会話で複数の話題を扱わず、話題ごとに会話を分ける。目安として、30分以上同じ会話を続けている場合は区切りを検討。

Q2: AIが嘘をついている気がする

原因: AIは「もっともらしい回答」を生成する仕組みです。知らないことでも、それらしい答えを作ってしまうことがあります。これをハルシネーション（幻覚）と呼びます。

対処法

数字・日付・人名・URLは必ず自分で確認する。「この情報の出典はありますか？」と聞く。重要な判断に使う情報は別の情報源でもクロスチェック。

大切な心構え: AIの回答は「優秀だが経験の浅いアシスタントの提案」だと思ってください。そのまま使うのではなく、あなたの判断で確認・修正して使います。

Q3: 何を頼んでいいかわからない

あなたの状況AIに頼めること
長い文書を読む時間がない「この文書を要約して。要点を5つにまとめて」
メールの文面が思いつかない「○○さんへの△△の件のメールを下書きして」
データを整理したい「このデータを表形式に整理して」
報告書の構成を考えたい「△△の報告書の目次案を3パターン作って」
文章を推敲したい「この文章をより読みやすく修正して」
アイデアを広げたい「△△について異なる視点から5つアイデアを出して」
英語の文書を読みたい「この英文を日本語に翻訳して。専門用語は原語も併記」

Q4: エラーが出た / 止まった

症状対処法
「エラー」と表示されたメッセージを読んで、指示を言い換えて再送信
途中で止まった「続けて」と送信する
同じ間違いを繰り返す新しいセッションで、指示を変えて再挑戦
ファイルが見つからないと言われる「このフォルダの中身を見せて」と頼む

大切なこと: エラーはあなたの能力不足ではありません。AIとの対話は試行錯誤が当たり前です。言い方を変えて、もう一度試してみてください。

Q5: 前に話した内容を覚えていない

原因: AIは会話（セッション）が切れると、前回の内容を忘れます。

対処法

重要な前提は毎回の会話冒頭で伝える。継続的に使う情報は CLAUDE.md に書いておく。前回の会話のまとめを最初に貼り付けてから新しい依頼をする。

第6章

上手な指示の作り方

良い指示の4つの要素

AIへの指示は、以下の4要素を含めると格段に良くなります。全部入れる必要はありませんが、意識するだけで結果が変わります。

指示の4要素

1. 役割
どんな立場で考えてほしいか
「あなたはワークショップのファシリテーターです」
2. 文脈
背景情報・状況説明
「来月、企業30社が参加する対話イベントを開催します」
3. 制約
守ってほしい条件
「所要時間は3時間以内で」「参加者は初対面同士です」
4. 形式
出力の形をどうしてほしいか
「タイムテーブル形式で」「箇条書きではなく文章で」

第7章

アプリを作る: 準備編

この章では、Claude Codeでアプリを作るまでの準備を、実際の操作手順に沿って説明します。一つずつ順番に進めてください。

ステップ1: Cursorをインストールする

  1. ブラウザで cursor.com にアクセスする
  2. 「Download」ボタンをクリック（Mac版が自動で選ばれる）
  3. ダウンロードされたファイルを開き、アプリケーションフォルダにドラッグ
  4. Cursorを起動し、アカウントを作成（Googleアカウントでログイン可能）
  5. 初回設定で「Claude」を選択する画面が出たら、Claudeを選ぶ
Cursorとは

AIと対話しながらファイルの作成・編集ができるアプリです。ChatGPTのようなチャット画面が右側にあり、左側でファイルの中身を確認できます。

ステップ2: 作業フォルダを作る

  1. Finderを開き、自分のホームフォルダ（家のアイコン）に移動
  2. projects というフォルダを作る（既にあればそのまま）
  3. その中にアプリ用のフォルダを作る（例: expense-checker）
  4. Cursorに戻り、メニューから「File」→「Open Folder」で今作ったフォルダを開く
フォルダ名のルール

フォルダ名は半角英数字とハイフンだけで付けてください。日本語やスペースが入ると不具合の原因になります。例: expense-checker（OK）、経費チェッカー（NG）

ステップ3: Claude Codeに話しかける

  1. Cursor画面の右側にチャットパネルが表示されている
  2. ここに日本語でやりたいことを入力する
  3. AIがファイルの作成・編集を提案してくる
  4. 「Accept」（承認）ボタンで変更を適用する
最初に送るメッセージの例
経費管理ツールをWebアプリとして作ってください。

【機能】
- 経費の入力（日付、カテゴリ、金額、メモ）
- カテゴリ別の集計
- CSV出力

1つのHTMLファイルで完結させてください。
データはブラウザのlocalStorageに保存して。

ステップ4: 作ったアプリを確認する

AIがファイルを作成したら、ブラウザで動作を確認します。

HTMLファイル1つで完結するアプリの場合

  1. Cursorの左側のファイル一覧で index.html を右クリック
  2. 「Open in Browser」またはファイルをFinderから直接ダブルクリック
  3. ブラウザでアプリが表示される

サーバーが必要なアプリの場合

AIが「サーバーを起動します」「http://localhost:5210 で確認できます」と言った場合:

  1. AIの指示通りにサーバー起動コマンドを実行する（AIがやってくれることが多い）
  2. ブラウザで指定のURL（例: http://localhost:5210）を開く
  3. アプリが表示される。このURLは自分のPCだけで見える状態
localhost とは

「自分のパソコンの中だけで動いているWebサイト」のことです。この段階では他の人は見れません。問題なければ次のステップで公開します。

ステップ5: 修正を依頼する

確認して気になる点があれば、そのままチャットで伝えます。何度でも修正できます。

修正依頼の例
いくつか修正をお願いします。
- 金額の入力欄に3桁ごとのカンマを表示して
- カテゴリに「接待費」を追加して
- 全体の文字サイズをもう少し大きくして
- ヘッダーの色を黒に変えて

ステップ6: インターネットに公開する

アプリが完成したら、AIに「公開して」と伝えるだけです。AIが以下を自動で行います。

AIがやること何が起きているか
GitHubにコードを保存ファイル一式をクラウド上のDropboxのような場所にバックアップ
Vercelにデプロイアプリをインターネット上のサーバーに配置
URLを発行https://あなたのアプリ名.vercel.app というURLが作られる
公開を依頼するメッセージ
完成です。GitHubに保存して、Vercelで公開してください。

公開が完了すると、AIが https://○○.vercel.app のようなURLを教えてくれます。このURLをSlackやメールで共有すれば、誰でもアクセスできます。

公開 = 誰でも見られる状態

Vercelで公開したアプリは、URLを知っている人なら誰でもアクセスできます。社内限定にしたい場合は、URLを社内だけで共有してください。パスワード保護が必要な場合は、AIに「パスワード認証をつけて」と依頼してください。

操作の全体像

ゼロから公開までの流れ

1
フォルダを作って開く
Finderで新規フォルダ作成 → Cursorで「Open Folder」
2
AIに依頼する
チャットに「○○を作って」と日本語で入力
3
ブラウザで確認する
index.htmlを開く or localhostのURLをブラウザに入力

AIが言うこと何が起きているかあなたがやること
「GitHubにプッシュします」ファイルをクラウドにバックアップしている何もしなくてOK

実際にアプリを作る

「アプリを作る」と聞くと、プログラミングの専門知識が必要だと思うかもしれません。しかしClaude Codeを使えば、日本語で「こういうものが欲しい」と伝えるだけで、実際に動くWebアプリを作ることができます。

以下は、ミラツク/エッセンスで実際にClaude Codeを使って作られたアプリの実例です。すべて、プログラミングのコードを一行も手書きせずに完成しています。

アプリの作り方: 3つのパターン

作りたいものの複雑さに応じて、3つのパターンがあります。

パターン1: 「1ページ完結」の業務ツール（最も簡単）

一番手軽なのは、1つのHTMLファイルで完結するツールです。ブラウザで開くだけで使えて、サーバーも不要。経費計算、チェックリスト、フォーマット変換など、「Excelでやっていた作業をブラウザで」というイメージです。

実際の依頼例: 経費チェッカーを作る
経費管理ツールをWebアプリとして作ってください。

【機能】
- 経費の入力フォーム（日付、カテゴリ、金額、メモ）
- カテゴリ別の集計表
- 月別のグラフ表示
- CSV出力ボタン

【デザイン】
- シンプルで見やすいデザイン
- スマホでも使えるようにレスポンシブ対応

【技術的な条件】
- 1つのHTMLファイルで完結させる
- データはブラウザのlocalStorageに保存
  （サーバー不要で、同じブラウザなら次回も
  データが残る）
ポイント

「1つのHTMLファイルで完結」「localStorageに保存」と伝えると、サーバーの設定なしで動くツールが作れます。作ったファイルをダブルクリックするだけで使えます。

パターン2: データを扱う業務アプリ（中級）

データの蓄積・検索・管理が必要なアプリです。研究者のデータベース、イベント参加者の管理、調査結果の蓄積など。Pythonのサーバーとデータベースを使いますが、Claude Codeがすべて構築してくれます。

実際の依頼例: 登壇者管理システムを作る
オンラインイベントの登壇者を管理するWebアプリを
作ってください。

【背景】
ミラツクでは月に複数回のオンラインイベントを
開催しており、登壇者の情報管理が属人化しています。

【必要な機能】
- 登壇者の情報登録（氏名、所属、専門分野、
  連絡先、過去の登壇履歴）
- イベントごとの登壇者アサイン
- 検索・フィルタ機能（専門分野、所属で絞り込み）
- 登壇者へのメール一括送信テンプレート

【デザイン】
- esse-senseのデザインテイストに合わせて
  （黒・白・赤のシンプルなデザイン）
- 左にリスト、右に詳細の2ペインレイアウト

【ユーザー】
- 社内スタッフ3-5名が利用
- ITに詳しくない人でも直感的に使えること

パターン3: 公開Webサイト（応用）

社外に公開するダッシュボードやポータルサイトです。調査結果の公開ページ、イベント情報サイトなどが該当します。GitHub PagesやVercelで無料公開できます。

実際の会話例: 日程候補チェッカーができるまで

以下は、実際にアプリを作ったときの会話の流れです。プログラミングの知識は一切使っていません。

自分
チームメンバーの空いている日程を自動で見つけるツールが欲しい。Googleカレンダーの情報をもとに、全員が空いている時間帯を一覧表示して。

日程候補チェッカーを作成します。まず要件を確認させてください。（1）何名くらいのメンバーを想定していますか？（2）カレンダーのデータはどう取得しますか？...
自分
3-5名くらい。カレンダーはiCalのURLから読み込めるようにして。
Claude Code
承知しました。Pythonのバックエンド + HTML/CSSのフロントエンドで構築します。iCalのURLを登録すると、全員の予定を重ね合わせて空き時間を表示する仕組みにします。作成を開始します。
... AIがファイルを作成・コードを書く（数分） ...
Claude Code
完成しました。http://localhost:5211 で確認できます。メンバーの追加・iCal URLの登録・期間選択の機能を実装しました。
自分
いいね。空き時間の表示をもう少し見やすくして。あと、結果をSlackに送れる機能も追加して。
... AIが修正・機能追加（数分） ...
自分
完璧。公開して。

このように、「欲しいものを日本語で伝える → 確認する → フィードバックする → 公開する」というサイクルで、プログラミングの知識なしにアプリが完成します。

よくある質問

質問回答
プログラミングを全く知らなくても作れる？作れます。実際にこのマニュアル自体がその方法で作られています。必要なのは「何が欲しいか」を言葉で伝える力です。
作ったアプリにバグがあったら？「ここがおかしい」「この操作をするとエラーが出る」と伝えれば、AIが修正します。
デザインが気に入らなかったら？「もっとシンプルに」「色をこう変えて」「この部分をもっと大きく」と伝えれば調整できます。
作ったアプリの維持管理は？修正や機能追加が必要になったら、同じように日本語で依頼するだけです。コードを読む必要はありません。
費用はかかる？Claude Codeの利用料のみ。アプリの公開はGitHub Pages（無料）やVercel（無料枠あり）で可能です。
まずは小さく始める

最初から大きなアプリを作ろうとせず、「Excelで30分かかっている作業を自動化する」くらいの小さなツールから始めてみてください。小さな成功体験が、次のアイデアにつながります。
